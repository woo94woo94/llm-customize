*** GLOSSARY ***

=== [속성 이름 => 의미 설명] ===
"PJT코드" => 프로젝트 기본정보

"PJT명" => 프로젝트 기본정보

"실부문" => 프로트 수행부서 정보(사업실)

"그룹부" => 프로젝트 수행부서 정보(그룹부)

"섹션팀" => 프로젝트 수행부서 정보(섹션팀)

"PM직번" => 프로젝트 수행 책임자 정보

"PJTType" => 프로젝트의 업무 성격에 따라 구분됨
- 용역(확정), 통신(확정), 정비(확정), 단품(확정) : 운영성 사업의 매출로 년단위 계약에 따라 고정적으로 매출 발생(예 : 포스코 IT SLA 및 EIC정비 사업, 그룹사 SM사업)
- 용역(진행),시스템(진행) : 개발성 사업매출로 비정기적으로 발생함)
※ XX(확정), XX(진행) Type은 월 단위 매출 인식 방법이 다름
   - XX(확정) : 고객사 청구 세금계산서 기준
   - XX(진행) : 원가진행률( 계약금액 *) 당월누계예정원가/총예정원가)-전월누계매출

"PJT상태" => 프로젝트 진행 상태를 구분하는  값
영업(고객사와 계약 전 상태),계약(고객사 계약 후), 실행(고객사 계약 후 PM이 실행계획을 승인받은상태, 실행상태 이후 매출 및 원가 발생), 완료(프로젝트 완료), 종료(프로젝트 완료 후 PM이 결재권자에게 완료품의 승인 得)

"발주처구분비매출PJT유형" => R&D(자체) : 연구소 등 R&D개발 부서에서 신기술 확보를 수행하는 과제
R&D(수탁) : R&D부서에서 외부 및 사업실행부서와 함께 수행하는 과제
투자,(구)투자 : 당사에서 투자하여 진행하는 과제
A/S 하자보수 : 본사업 후, 하자보수를 위하여 등록하는 프로젝트
기타 : 기타(부서 및 프로젝트 단위 

"비매출PJT목적" => 비매출 PJT의 목적을 선택하는 항목
- 신제품개발 : 시장 출시를 목표로 한 신규 제품 개발
- Pre-R&D : 본격 R&D 전 사전 조사·기획 단계
- 신기술확보 : 신규 기술 도입·내재화를 위한 연구
-기능수정, 추가 : 기존 제품·시스템의 기능 개선 또는 추가
-Upgrade : 성능 향상, 버전 업그레이드 목적

"비매출PJT발의부서" => 비매출 프로젝트의 비용 부담부서

"비매출PJT활용부서" => 비매출 프로젝트의 비용 활용 부서

"고객구분" => 고객사를 포스코,그룹사,대외로 구분하는 속성으로 
분석 시, 포스코와 그룹사는 대내사업으로 구분함

"원발주처" => 프로젝트의 최초 발주자(원청), 즉 당사 고객의 상위 고객

"발주처" => 당사의 고객사

"PBS대" => 프로젝트 정보분류 체계(Project Breakdown Structures). 프로젝트의 사업분야 및 공장 설비단위까지 체계적으로 분류한 체계

"PBS중" => 프로젝트 정보분류 체계(Project Breakdown Structures). 프로젝트의 사업분야 및 공장 설비단위까지 체계적으로 분류한 체계

"PBS소" => 프로젝트 정보분류 체계(Project Breakdown Structures). 프로젝트의 사업분야 및 공장 설비단위까지 체계적으로 분류한 체계

"SBU코드" => 전략사업 단위 코드

"SBU명" => 전략사업 단위 명

"영업담당자" => 영업담당자

"계약구분" => 고객사와의 계약이 입찰 또는 수의계약 방식으로 체결되었는지 구분하는 항목

"영업상태" => 프로젝트의 영업상태를 구분하는 값
1.기회포착 : 영업 담당자가 잠재기회를 시스템에 등록한 상태
2.사전영업 : 영업 담당자가  영업기회를 시스템에 등록한 상태
3.견적 : 영업 담당자가 견적 담당자에게 시스템을 통하여 견적요청을 한 상태
4.입찰 : 수주심의 가결 상태
5.시담 : 계약심의 가결 상태
6.계약 : 고객과의 계약이 완료되어 최초계약수주인식을 영업부서장이 승인까지 완료된 상태
7.Drop(실패) : 수주 실패로 영업기회가 종료된 상태

"모PJT코드" => 포스코SLA서비스와 같이 연관프로젝트 관리를 위해 상위프로젝트 코드 입력

"수주심의대상여부" => 수주심의 대상여부를 판단하는 항목
※수주심의 : 사업의 리스크 및 수익성에 대해 심의를 하고 사업 참여여부를 결정하는 과정

"하도급발주심의" => 회사가 중점적으로 관리하는 프로젝트 여부를 판단하는 항목


"선실행여부" => 선실행여부를 판단하는 항목
※ 선실행 : 발주자와의 정식 수주계약 체결 ㅇ이전 또는 실행계획전에 긴급하게 하도급(구매포함)을 활용하여야 하기
             위해 운영되는 사업수행 프로세스

"중점관리여부" => 중점관리여부를 판단하는 항목
※ 당사는 '25년부터 중요성이 크거나 전략적 사업등을 선별하여 해당 사업의 리스크 중점관리 중

"NSC" => NSC사업여부를 판단하는 항목

  공급원가에 관리비를 반영하여 납품하도록 지정해주는 발주 방식

※채무보증: 프로젝트파이낸싱(Project Financing)에서 금융기관이 제공한 대출금 상환을 보증인이 대신 책임지겠다고 약정하는 행위
